# group-request-handler

MaiBot 群聊申请处理插件 — 监听群聊邀请，管理员审核 + 白名单管理 + 群列表查询。

## 功能

- **群邀请审核**：收到加群邀请时通过 Webhook 捕获，通知管理员审核（通过/拒绝）
- **白名单巡检**：定时检查 Bot 所在群列表，不在白名单的群自动进入审核队列并**静默消息**
- **消息静默**：审核中的群，Bot 入站/出站消息全部拦截，不会在该群发任何消息
- **群列表查询**：查看 Bot 当前所在群、群详情
- **退群管理**：手动退群或审核拒绝自动退群

## 架构

```
NapCat 群邀请事件
    │
    ▼
┌─────────────┐    Webhook     ┌─────────────────────┐
│  NapCat     │ ─────────────► │  group-request       │
│  HTTP 客户端  │  POST + JSON  │  -handler 插件        │
│  (上报)      │               │  HTTP Server 监听     │
└─────────────┘               └──────────┬──────────┘
                                         │
                              ┌──────────▼──────────┐
                              │  管理员私聊推送通知    │
                              │  白名单巡检 & 静默    │
                              │  管理命令处理         │
                              └─────────────────────┘
```

## 安装

将 `group-request-handler` 目录放入 MaiBot 的 `plugins/` 下，重启 MaiBot 即可。

### 依赖

- Python 3.10+
- `aiohttp >= 3.9.0`（由 MaiBot 包管理自动安装）

## NapCat 配置

插件通过 Webhook 接收群邀请事件，需要在 NapCat 的 **HTTP 客户端（HTTP Client）** 中配置事件上报。

### 1. NapCat HTTP 客户端配置

打开 NapCat 的 HTTP 客户端配置文件（一般在 NapCat 设置 → HTTP 客户端），添加或修改：

```json
{
  "http": {
    "enable": true,
    "host": "0.0.0.0",
    "port": 18081,
    "secret": "",
    "enableHeart": true,
    "enablePost": true,
    "postUrls": [
      "http://127.0.0.1:18081/maibot/group_request"
    ]
  }
}
```

- `port`：HTTP 客户端监听的端口，需与插件 `config.toml` 中的 `webhook.port` 一致（默认 18081）
- `postUrls`：事件上报地址，格式为 `http://<插件 host>:<插件 port><插件 path>`
- `secret`：如果填写了，插件 `webhook.secret` 也必须填写一致的值，用于签名校验

### 2. NapCat 需启用的权限/事件

确保 NapCat 启用以下事件上报（通常默认已开启）：

- `request.group.invite` — 群邀请事件

### 3. 常见问题

| 问题 | 检查 |
|------|------|
| 收不到群邀请通知 | NapCat HTTP 客户端 `enablePost` 是否为 true，`postUrls` 是否正确 |
| Webhook 校验失败 | NapCat 和插件的 `secret` 是否一致 |
| 插件端口被占用 | 修改 `webhook.port`，同时更新 NapCat 的 `postUrls` |

## 指令说明

所有指令仅限**管理员**（`admin.admin_qqs` 中配置的 QQ 号）使用。

### 群聊管理

| 指令 | 说明 | 示例 |
|------|------|------|
| `/群管理` | 查看所有管理指令帮助 | `/群管理` |
| `/群列表` | 查看 Bot 当前所在的所有群，显示群名、群号、人数、白名单/审核状态 | `/群列表` |
| `/群信息 <群号>` | 查看指定群的详细信息（群名、人数、群主、群介绍、白名单/静默状态） | `/群信息 123456789` |
| `/退群 <群号>` | 机器人退出指定群聊 | `/退群 123456789` |

### 群邀请审核

| 指令 | 说明 | 示例 |
|------|------|------|
| `/同意 <群号>` | 同意群邀请，Bot 加入群聊 | `/同意 123456789` |
| `/拒绝 <群号>` | 拒绝群邀请（配置了 `leave_on_invite_reject=true` 时同时退群） | `/拒绝 123456789` |

### 群审核（白名单巡检触发）

当 Bot 所在群不在白名单中，管  理员会收到私聊推送，然后：

| 指令 | 说明 | 示例 |
|------|------|------|
| `/审核通过 <群号>` | 通过该群审核，Bot 恢复正常发言，**自动加入白名单** | `/审核通过 123456789` |
| `/审核拒绝 <群号>` | 拒绝该群，Bot 退出群聊 | `/审核拒绝 123456789` |

### 白名单管理

| 指令 | 说明 | 示例 |
|------|------|------|
| `/白名单` | 查看当前白名单中的所有群 | `/白名单` |
| `/加白 <群号>` | 将群添加到白名单（同时解除该群的消息静默） | `/加白 123456789` |
| `/删白 <群号>` | 将群从白名单中移除 | `/删白 123456789` |

## 配置说明

复制 `config.example.toml` 为 `config.toml`，按需修改。

```toml
[plugin]
enabled = true              # 是否启用插件

[admin]
admin_qqs = ["你的QQ号"]     # 管理员 QQ 列表，群邀请通知推送到这些 QQ 私聊

[webhook]
host = "127.0.0.1"          # Webhook 监听地址
port = 18081                # 监听端口，需与 NapCat HTTP 客户端上报端口一致
path = "/maibot/group_request"  # HTTP 路径，需与 NapCat postUrls 路径一致
secret = ""                 # 签名密钥，留空则不校验；填写则需与 NapCat HTTP 客户端 token 一致

[whitelist]
enforce_whitelist = true          # 开启白名单强制：不在白名单的群自动进入审核
group_whitelist = ["群号1", "群号2"]  # 白名单群号列表，空 = 不限制
check_interval_seconds = 60       # 白名单巡检间隔（秒），最小 10 秒
leave_on_invite_reject = true     # 拒绝群邀请后，如果在群内则自动退群
```

## 工作流程

### 群邀请流程

1. 有人邀请 Bot 加群 → NapCat 推送 `request.group.invite` 事件到插件 Webhook
2. 插件通知所有管理员（QQ 私聊），附带群号、群名、邀请人、验证消息
3. 管理员回复 `/同意 <群号>` 或 `/拒绝 <群号>` 处理邀请

### 白名单巡检流程

1. 定时（`check_interval_seconds`）获取 Bot 当前群列表
2. 不在白名单的群 → 加入审核队列，Bot 消息自动静默，推送审核通知给管理员
3. 管理员 `/审核通过 <群号>` → 解除静默 + 自动加入白名单
4. 管理员 `/审核拒绝 <群号>` → Bot 退出该群
5. 冷却期 10 分钟内同一群不会重复推送

### 消息静默说明

审核中的群，Bot **不会发送也不会响应任何消息**，但管理命令（/审核通过、/审核拒绝 等）在审核群里依然可以执行。管理员私聊的所有命令不受静默影响。

## 设计说明

### 私聊通知为何绕过 SDK 消息管线

插件的管理通知（群邀请提醒、白名单巡检告警等）通过 `api.call` 直接调用底层 API 发送私聊消息，而非走 MaiBot SDK 的标准 `send.text` 管线。原因：

- **避免被其他插件误拦截**：审核类插件的管理通知属于关键安全通知，必须确保送达。标准 SDK 消息管线可能被其他插件（如静默、过滤、拦截类插件）中途截获或丢弃，导致管理员收不到审核提醒。
- **不受当前群静默状态影响**：白名单巡检会对审核中的群做消息静默，但管理通知需要旁路这个限制，直接送达管理员私聊。
- **解耦通知通道与聊天通道**：管理通知和群聊消息是两个不同语义的通道，走独立路径更清晰，也方便后续维护者区分哪些是"管理通知"、哪些是"群聊回复"。

此设计权衡在审核类插件中属于合理做法，后续如果你作为维护者需要统一消息通知出口，可在此处将 `_send_private_text` 改为注入一个可配置的通知策略接口。
